\# Instructions for AI Agents



Before making any change:



1\. Read PROJECT\_CONTEXT.md.

2\. Read ARCHITECTURE.md.

3\. Read TODO.md.



Never:



\- rewrite working systems

\- introduce unnecessary abstractions

\- replace code without explanation

\- remove debugging helpers unless requested



Always:



\- preserve backward compatibility

\- prefer incremental refactoring

\- explain architectural decisions

\- update TODO.md after completing work

\- update ARCHITECTURE.md when architecture changes



Coding style:



\- modern ES modules

\- descriptive variable names

\- no unnecessary comments

\- keep functions small

\- avoid duplicated logic



When uncertain:



Ask instead of guessing.



Priority:



Working code >

Clean code >

Fancy code



The project owner prefers:



\- practical solutions

\- incremental improvements

\- minimal dependencies

\- readable code over clever code
